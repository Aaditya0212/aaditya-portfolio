Prompt Performance & User Behavior Dashboard

This Power BI dashboard was built to analyze how users interact with a prompt-based system tracking prompt success rate, engagement, user satisfaction, and behavior across user types, plans, and device usage.

Power BI analytics dashboard

One of the key goals of this project was to measure performance across different prompt versions (v1 and v2), simulate A/B testing results, and surface insights that could help product teams and analysts make informed decisions.

📊 Dashboard Overview

The main view brings together overall system performance metrics and user behavior patterns.

📈 Key KPIs shown on the dashboard:

User Satisfaction: 3.79/5 • Success Rate: 91% • Avg Prompts per Session: 1.02 • Avg Response Time: 2.48 sec • Top Device: Web (59.44%) • Peak Day: Tuesday

The layout includes filters for user type, plan (free/premium), prompt version, and time period, which allows quick segmentation across all charts and KPIs.

🧪 A/B Testing Prompt Versions (v1 vs v2)

To evaluate the performance of two different prompt versions, the dashboard includes a filter on the prompt_version field. Using this, we simulated an A/B testing scenario by isolating data for v1 and v2.

Below is a side-by-side comparison using the dashboard filters:

While both versions showed similar success rates (~90%), v2 had a slightly higher average satisfaction score and reduced prompt count per session—suggesting a small improvement in overall interaction quality.

💼 Segmented Analysis: Premium Users

By filtering for Premium users, the dashboard reveals slightly different behavioral trends:

🔍 Key Findings:

  • Premium users had a higher success rate and satisfaction score
  • They submitted fewer prompts on average, but the interaction quality was better
  • Feedback skewed more positively in comparison to free users
  • Web usage dominates (~60%), Mobile has significant share (~31%)
  • Activity drops on weekends and peaks mid-week

⚙️ Tools Used

  • Power BI for dashboarding
  • DAX for KPI logic and calculated measures
  • Excel for base data handling
  • Prompt version filtering to simulate A/B testing
  • Slicers for plan, user type, device, and time

💡 Conclusion

This dashboard wasn't just built for visualization—it was built to answer specific business questions and simulate how changes in prompt versions or user segments affect engagement. It serves as a compact analytics tool for product teams measuring feature changes, analysts running segmentation or A/B testing, and business leads reviewing high-level usage KPIs.
